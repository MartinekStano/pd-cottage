= Zadanie pre Codex - Pallov Dvor
:toc:
:sectnums:

== Kontext

Tento dokument je hlavne technicke a implementacne zadanie pre frontend webu `Pallov Dvor`.

Pallov Dvor je sukromna chata v obci Lutise, Ulica 373. Web ma prezentovat pokojne kratkodobe ubytovanie v sukromi, nie hotel, penzion, restauraciu ani wellness rezort.

Detailny obsah jednotlivych stranok je rozdeleny v priecinku `instruction/pages/`. Globalne pravidla su v `instruction/shared/`.

== Technicke poziadavky

* Pouzi existujuci projekt v Next.js.
* Projekt pouziva Next.js `16.2.4`, React `19.2.4`, TypeScript a Tailwind CSS `4`.
* Pouzi App Router.
* Pred upravami Next.js kodu precitaj relevantnu dokumentaciu v `node_modules/next/dist/docs/`, pretoze aktualna verzia ma zmeny oproti starsim znalostiam.
* Web ma byt bez backendu; rezervacny formular moze byt staticky frontend prototyp.
* CTA prvky maju viest na kontakt alebo formular s dopytom.
* Kod rozdel na znovupouzitelne komponenty a datove struktury.
* Texty a opakovane hodnoty drz centralizovane, aby sa neskor dali lahko menit.
* Pridaj SEO metadata pre kazdu podstranku.
* Web musi byt responzivny pre mobil aj desktop.
* Na mobile musia byt formularove polia, CTA tlacidla, tabulky cien a karty citatelne bez horizontalneho scrollu, okrem pripadov, kde je scroll tabulky explicitne lepsie riesenie.

== Fotografie a assety

Do repozitara patria optimalizovane a pomenovane kópie v `public/images/pallov-dvor/`.
Atrakcie maju pripravene fotky v `public/images/attractions/`.
Surove importy fotiek nie su sucastou commitu. Mapovanie optimalizovanych suborov na povodne nazvy importu je v `public/images/pallov-dvor/manifest.json`.

Pri implementacii vyber realne fotografie z tychto kategorii v optimalizovanych assetoch:

* `Exterier Chata`
* `Chata 1. obytna jednotka`
* `Chata 2. obytna jednotka`
* `Spolocny priestor`

Ak fotka pre sekciu este nie je vybrana, priprav stabilny image placeholder a nazov suboru podla prislusneho page zadania.

== UI smer

Web ma posobit moderne, pokojne, vzdusne, prirodne a doveryhodne. Vizualny detail je v `instruction/shared/brand-ui.md`.

Preferuj:

* pokojnu prirodnu paletu,
* citatelnu typografiu,
* vyrazne CTA,
* prehladne karty pre opakovane informacie,
* dostatok priestoru medzi sekciami,
* jednoduche animacie a hover stavy iba tam, kde pomahaju orientacii.

== Implementacny postup

1. Precitaj `AGENTS.md` a relevantne Next.js docs v `node_modules/next/dist/docs/`.
2. Navrhni centralne data pre navigaciu, kontakt, ceny, vybavenie a obsah stranok.
3. Priprav layout, hlavicku, paticku a spolocne UI komponenty.
4. Implementuj podstranky podla `instruction/pages/`.
5. Pouzi realne fotografie alebo priprav image placeholdery.
6. Over `npm run lint` a `npm run build`, ak to prostredie dovoli.
7. Spusti dev server a skontroluj web na mobile aj desktope.
